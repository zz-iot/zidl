# zidl — Testing Guide

---

## Test Suites

### Unit and golden tests

```sh
zig build test
```

Runs:
- **Library unit tests** (678): parser, lexer, preprocessor, semantic analysis, IR builder, backend codegen, and backend interface utilities.
- **CLI unit test** (1): smoke test of the CLI entry point.
- **Zig integration tests** (8): compile the committed `test/golden/zig/types.zig` against `zidl-rt` and run CDR round-trip and vtable tests.
- **Golden output comparison**: regenerates all backends into `build-tmp/golden-check/` and compares byte-for-byte against `test/golden/`. Any added, removed, or changed file is a failure.

Also runs the package tests:

```sh
# run separately, or they're triggered by CI
zig build test --build-file packages/zidl-rt/build.zig    # 61 tests
zig build test --build-file packages/zidl-cdr/build.zig   # 44 tests
```

Or from the package directory:

```sh
cd packages/zidl-rt  && zig build test   # 61 CDR round-trip tests
cd packages/zidl-cdr && zig build test   # 44 CDR cross-validation tests
```

### Integration tests

```sh
zig build integration-test
```

Compiles and runs the generated C, C++, and Java code against the committed golden files:

- **C**: links `test/golden/c/types_cdr.c` + `packages/zidl-cdr/src/zidl_cdr.c`, compiles with `-std=c99 -Wall -Werror` using Zig's bundled clang. No external toolchain needed.
- **C++**: links `test/golden/cpp/types_cdr.cpp` + `zidl_cdr.c`, compiles with `-std=c++17` using Zig's bundled clang++. No external toolchain needed.
- **Java**: compiles `test/golden/java/Types.java` + `test/integration/java/Test.java` with `javac`, then runs with `java`. Skipped with a warning if `javac`/`java` is not on `PATH`.

### CDR interop tests

```sh
zig build interop-test
```

Runs 10 Zig tests against committed CDR byte vectors in `interop/zig_interop_test.zig`. No Cyclone DDS installation required — the expected bytes are hardcoded and were generated by the Cyclone DDS harness.

## Golden Files

### What they are

`test/golden/types.idl` is the canonical test IDL. zidl generates output from it for all backends in two modes:

| Directory | Mode |
|---|---|
| `test/golden/zig/` | Zig, single-file |
| `test/golden/zig-split/` | Zig, `--split-files` |
| `test/golden/c/` | C, single-file |
| `test/golden/c-split/` | C, `--split-files` |
| `test/golden/cpp/` | C++, single-file |
| `test/golden/cpp-split/` | C++, `--split-files` |
| `test/golden/java/` | Java, single-file |
| `test/golden/java-split/` | Java, `--split-files` |

All files in these directories are committed and checked into git. They serve two purposes:
1. Regression detection — `zig build test` fails if any output changes.
2. Readability reference — reading a golden file shows exactly what a user would get for a non-trivial IDL.

### Regenerating golden files

After intentional changes to a backend, regenerate and commit:

```sh
zig build regen-goldens
git diff test/golden/         # review the diff
git add test/golden/ && git commit -m "..."
```

`regen-goldens` runs all four backends in both single-file and split-file modes and writes directly into `test/golden/`.

### Reviewing a diff

`zig build check-goldens` regenerates into `build-tmp/golden-check/` and compares with `check_goldens` (a bidirectional directory diff tool). The check is integrated into `zig build test` — a mismatch fails the build. To inspect the diff without failing:

```sh
diff -r test/golden/c build-tmp/golden-check/c
```

## Cyclone DDS Interop Harness

The `interop/` directory contains the harness used to generate the committed CDR byte vectors. Running it requires a Cyclone DDS checkout.

### Setup

```sh
make -C interop regen CYCLONE=/path/to/cyclonedds
```

This compiles zidl-generated C code alongside Cyclone DDS, serializes sample values with Cyclone, and writes the resulting byte arrays into `interop/zig_interop_test.zig`. Commit the updated file to record the new expected values.

### What's validated

The committed byte vectors test:
- Encap header byte values (`CDR1_LE = 0x0001`, `CDR2_LE = 0x0007`).
- Primitive type encoding (all IDL primitive types).
- String encoding (length-prefixed, NUL-terminated).
- Struct member layout and padding.
- `@appendable` DHEADER framing.
- TypeObject EquivalenceHash byte values (confirmed matching Cyclone's computed hash).

### Running without Cyclone

The committed byte vectors in `interop/zig_interop_test.zig` are sufficient for day-to-day development:

```sh
zig build interop-test   # no Cyclone needed
```

Only regenerate if `types.idl` or `interop/cyclone_dump.c` changes.

## zidl-rt / zidl-cdr Cross-Validation

`packages/zidl-cdr/` includes 44 tests that validate the C99 CDR library against `zidl-rt` (Zig) byte-for-byte. This catches encoding divergence between the two runtimes — generated C code uses `zidl-cdr`, generated Zig code uses `zidl-rt`, and they must produce identical wire bytes.

Coverage includes: all primitive types, strings, sequences, DHEADER framing for `@appendable`, PL_CDR EMHEADER framing for `@mutable`, and key hash computation.

## XRCE MicroZig Fixture

The XRCE profile test fixture in `test/xrce-microzig/` is compiled against Zig 0.15.1 (MicroZig compatibility). It is regenerated and compiled in CI but is not part of `zig build test`:

```sh
# Regenerate (requires Zig 0.16.0):
zig build run -- -b zig --profile xrce --zig-version 0.15.1 \
  -o test/xrce-microzig/generated test/xrce-microzig/types.idl

# Compile (requires Zig 0.15.1):
cd test/xrce-microzig && zig build test
```
